Tapix dispatches events at key points during the import lifecycle. You can listen to these events to trigger notifications, log activity, update dashboards, or perform any custom logic.

All events use Laravel's `Dispatchable` trait and can be listened to via `Event::listen()`, in your `AppServiceProvider`, or through [Laravel's automatic event discovery](https://laravel.com/docs/events#event-discovery).

## Import-Level Events

These events fire once per import at the start and end of the process.

```php
use Tapix\Core\Events\ImportStarting;
use Tapix\Core\Events\ImportStarted;
use Tapix\Core\Events\ImportCompleted;
use Tapix\Core\Events\ImportFailed;
```

### ImportStarting

Dispatched before the import begins processing, just before the `beforeImport` lifecycle hook runs. Use this for setup tasks that should happen before any importer logic.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |

### ImportStarted

Dispatched when `ExecuteImportJob` begins processing an import, before rows are iterated.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |

### ImportCompleted

Dispatched when the import finishes successfully with all rows processed.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |

### ImportFailed

Dispatched when the import fails due to an unrecoverable exception.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |

## Row-Level Events

These events fire for individual rows during processing, giving you fine-grained control over the import lifecycle.

```php
use Tapix\Core\Events\ImportRowProcessing;
use Tapix\Core\Events\ImportRowProcessed;
use Tapix\Core\Events\ImportRowCreating;
use Tapix\Core\Events\ImportRowCreated;
use Tapix\Core\Events\ImportRowUpdating;
use Tapix\Core\Events\ImportRowUpdated;
use Tapix\Core\Events\ImportRowSkipped;
use Tapix\Core\Events\ImportRowFailed;
```

### ImportRowProcessing

Dispatched before each row is processed. Use this for per-row logging or to prepare external state.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number being processed. |
| `$data` | `array` | The row data. |

### ImportRowCreating

Dispatched before a new record is saved to the database. The model instance has been instantiated but not yet persisted.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number being processed. |
| `$data` | `array` | The prepared row data. |
| `$record` | `Model` | The unsaved model instance. |

### ImportRowCreated

Dispatched after a new record is saved. The model has a primary key and exists in the database.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number that was processed. |
| `$data` | `array` | The prepared row data. |
| `$record` | `Model` | The saved model instance. |

### ImportRowUpdating

Dispatched before an existing record is updated.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number being processed. |
| `$data` | `array` | The prepared row data. |
| `$record` | `Model` | The existing model instance. |

### ImportRowUpdated

Dispatched after an existing record is updated and saved.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number that was processed. |
| `$data` | `array` | The prepared row data. |
| `$record` | `Model` | The updated model instance. |

### ImportRowSkipped

Dispatched when a row is skipped due to unresolved validation errors, missing match records, or being explicitly marked as skipped.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number that was skipped. |
| `$data` | `array` | The row data. |

### ImportRowFailed

Dispatched when a row fails processing due to an exception.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$rowNumber` | `int` | The row number that failed. |
| `$data` | `array` | The raw row data. |
| `$error` | `string` | The error message (truncated to 500 characters). |

### ImportRowProcessed

Dispatched after each chunk of rows is processed during `ExecuteImportJob`. Use this event for real-time progress tracking.

| Property | Type | Description |
|----------|------|-------------|
| `$import` | `Import` | The import model instance. |
| `$processedCount` | `int` | Number of rows processed so far. |
| `$totalCount` | `int` | Total number of rows in the import. |

## Event Dispatch Order

For each row, events are dispatched in this order:

1. `ImportRowProcessing` -- before any processing
2. `ImportRowCreating` or `ImportRowUpdating` -- before the database write
3. `ImportRowCreated` or `ImportRowUpdated` -- after the database write
4. `ImportRowProcessed` -- after the entire chunk completes

If a row is skipped, only `ImportRowProcessing` and `ImportRowSkipped` fire. If a row fails, `ImportRowProcessing` and `ImportRowFailed` fire.

## Listener Example

Create a listener to respond to import completion:

```php [app/Listeners/SendImportReport.php]
use Tapix\Core\Events\ImportCompleted;

class SendImportReport
{
    public function handle(ImportCompleted $event): void
    {
        $import = $event->import;

        logger()->info("Import completed", [
            'id' => $import->id,
            'created' => $import->created_rows,
            'updated' => $import->updated_rows,
            'skipped' => $import->skipped_rows,
            'failed' => $import->failed_rows,
        ]);
    }
}
```

Register the listener in your `AppServiceProvider` or use [Laravel's automatic event discovery](https://laravel.com/docs/events#event-discovery).

## Auditing Row Changes

The granular row events are useful for building an audit trail of every record created or updated by an import:

```php [app/Listeners/AuditImportedRow.php]
use Tapix\Core\Events\ImportRowCreated;

class AuditImportedRow
{
    public function handle(ImportRowCreated $event): void
    {
        AuditLog::create([
            'import_id' => $event->import->id,
            'row_number' => $event->rowNumber,
            'record_type' => $event->record->getMorphClass(),
            'record_id' => $event->record->getKey(),
            'action' => 'created',
        ]);
    }
}
```

## Progress Tracking

The `ImportRowProcessed` event enables real-time progress tracking. Since it fires after each chunk, you can calculate the percentage and broadcast updates to the frontend:

```php [app/Listeners/TrackImportProgress.php]
use Tapix\Core\Events\ImportRowProcessed;

class TrackImportProgress
{
    public function handle(ImportRowProcessed $event): void
    {
        $percentage = ($event->processedCount / $event->totalCount) * 100;

        broadcast(new ImportProgressUpdated(
            importId: $event->import->id,
            percentage: round($percentage, 1),
            processed: $event->processedCount,
            total: $event->totalCount,
        ));
    }
}
```

This pairs well with a frontend component that listens for broadcast events to display a live progress bar during import execution.
