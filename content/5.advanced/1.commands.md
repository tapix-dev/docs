Tapix provides two Artisan commands: one for scaffolding new importers and one for cleaning up old import data.

## make:tapix-importer

Generates a new importer class in `app/Importers/`.

```bash [Terminal]
php artisan make:tapix-importer ContactImporter
```

### Options

| Flag | Short | Description |
|------|-------|-------------|
| `--model=` | `-m` | Specify the target model class. Accepts a fully qualified class name or a short name resolved under `App\Models`. |
| `--generate` | `-g` | Auto-generate fields from the model's database table columns. Requires `--model`. |
| `--children` | `-c` | Include a `children()` method stub for multi-entity imports. |

### Basic Usage

```bash [Terminal]
# Generate with default stub
php artisan make:tapix-importer ContactImporter

# Specify the target model
php artisan make:tapix-importer ContactImporter --model=Contact

# Auto-generate fields from the model's table schema
php artisan make:tapix-importer ContactImporter --model=Contact --generate

# Include a children() method stub
php artisan make:tapix-importer CompanyImporter --model=Company --generate --children
```

### Generated Output

The basic stub extends `BaseImporter` with `model()` and `fields()` methods:

```php [app/Importers/ContactImporter.php]
namespace App\Importers;

use Tapix\Core\Fields\ImportField;
use Tapix\Core\Importing\BaseImporter;

final class ContactImporter extends BaseImporter
{
    public function model(): string
    {
        return \App\Models\Contact::class;
    }

    public function fields(): array
    {
        return [
            ImportField::make('name')->required(),
        ];
    }
}
```

With `--generate`, fields are inferred from database columns. Column types are mapped to appropriate `ImportField` factory methods (`currency()`, `date()`, `boolean()`, `belongsTo()`), and columns like `id`, `created_at`, `updated_at`, `deleted_at`, and `tenant_id` are automatically excluded.

With `--children`, a `children()` method stub is added with a commented-out `ChildEntity` example.

::note
When using `--model` without a full namespace, the command resolves the class under `App\Models\`. For example, `--model=Contact` resolves to `App\Models\Contact`.
::

## tapix:cleanup

Removes old import records from the database. Runs two cleanup passes:

```bash [Terminal]
php artisan tapix:cleanup
```

### Completed and Failed Imports

Deletes imports with a status of `Completed` or `Failed` that are older than the specified number of days. Defaults to the `cleanup_after_days` config value (30 days).

```bash [Terminal]
php artisan tapix:cleanup --days=60
```

### Abandoned Imports

Deletes imports in non-terminal statuses (`Uploading`, `Mapping`, `Reviewing`, `Previewing`, `Importing`) that are older than the specified number of hours. Defaults to 24 hours.

```bash [Terminal]
php artisan tapix:cleanup --hours=48
```

### Orphaned Import Stores

The cleanup command also removes orphaned import store directories from the filesystem. These are SQLite store directories under the configured `store.path` that no longer have a corresponding `Import` record in the database.

### Scheduling

Run the cleanup command daily to keep the database tidy:

```php [routes/console.php]
Schedule::command('tapix:cleanup')->daily();
```

::note
Schedule the cleanup command in `routes/console.php` to run daily. This prevents the `tapix_imports` table and import store directories from growing indefinitely.
::

### Failed Import Rows

The `FailedImportRow` model uses Laravel's `MassPrunable` trait. Failed rows older than the `cleanup_after_days` config value are automatically included in the prunable query. Clean them up with:

```bash [Terminal]
php artisan model:prune
```

You can schedule this alongside the cleanup command:

```php [routes/console.php]
Schedule::command('tapix:cleanup')->daily();
Schedule::command('model:prune')->daily();
```
