This guide covers common patterns for testing Tapix importers using Pest PHP.

## Testing Importers

Test your importer's field definitions directly by instantiating the class:

```php
use App\Importers\ContactImporter;

it('defines required fields', function () {
    $importer = new ContactImporter();
    $fields = $importer->fields();

    expect($fields)->toHaveCount(5);
    expect($fields->get('email'))->not->toBeNull();
    expect($fields->get('email')->required)->toBeTrue();
});
```

## Testing with Models

Use the `Import` model and `ImportStore` to set up import state in your tests. Import rows live in per-import SQLite files, not in the main database.

```php
use App\Importers\ContactImporter;
use App\Models\User;
use Tapix\Core\Enums\ImportStatus;
use Tapix\Core\Models\Import;
use Tapix\Core\Store\ImportStore;

beforeEach(function () {
    $this->user = User::factory()->create();
});

it('tracks import progress', function () {
    $import = Import::create([
        'user_id' => $this->user->id,
        'importer' => ContactImporter::class,
        'file_name' => 'contacts.csv',
        'status' => ImportStatus::Uploading,
        'headers' => ['name', 'email'],
        'column_mappings' => [],
        'total_rows' => 100,
    ]);

    expect($import->status)->toBe(ImportStatus::Uploading);
});

it('can create and query import rows', function () {
    $import = Import::create([
        'user_id' => $this->user->id,
        'importer' => ContactImporter::class,
        'file_name' => 'contacts.csv',
        'status' => ImportStatus::Reviewing,
        'headers' => ['name', 'email'],
        'column_mappings' => [],
        'total_rows' => 1,
    ]);

    $store = ImportStore::create($import->id);

    $store->query()->insert([
        'row_number' => 1,
        'raw_data' => json_encode(['name' => 'John', 'email' => 'john@example.com']),
        'validation' => json_encode([]),
        'corrections' => json_encode([]),
        'skipped' => json_encode([]),
    ]);

    expect($store->query()->count())->toBe(1);
    expect($store->query()->first()->getFinalValue('name'))->toBe('John');
});
```

## Event Assertions

Verify that Tapix dispatches the correct events during the import lifecycle:

```php
use App\Importers\ContactImporter;
use App\Models\User;
use Illuminate\Support\Facades\Event;
use Tapix\Core\Enums\ImportStatus;
use Tapix\Core\Events\ImportCompleted;
use Tapix\Core\Models\Import;

it('dispatches ImportCompleted event', function () {
    Event::fake([ImportCompleted::class]);

    $user = User::factory()->create();
    $import = Import::create([
        'user_id' => $user->id,
        'importer' => ContactImporter::class,
        'file_name' => 'contacts.csv',
        'status' => ImportStatus::Importing,
        'headers' => [],
        'column_mappings' => [],
        'total_rows' => 0,
    ]);

    // ... trigger import execution ...

    Event::assertDispatched(ImportCompleted::class, function ($event) use ($import) {
        return $event->import->id === $import->id;
    });
});
```

## Queue Testing

Assert that the correct jobs are dispatched when an import is triggered:

```php
use Illuminate\Support\Facades\Queue;
use Tapix\Core\Jobs\ExecuteImportJob;

it('dispatches import job', function () {
    Queue::fake([ExecuteImportJob::class]);

    // ... trigger import ...

    Queue::assertPushed(ExecuteImportJob::class);
});
```

## Testing Lifecycle Hooks

Test custom lifecycle hooks by creating an anonymous importer class:

```php
use Illuminate\Database\Eloquent\Model;
use Tapix\Core\Importing\BaseImporter;

it('runs beforeSave hook', function () {
    $importer = new class extends BaseImporter {
        public bool $hookCalled = false;

        public function model(): string { return \App\Models\Contact::class; }
        public function fields(): array { return []); }

        public function beforeSave(Model $model, array $data): void
        {
            $this->hookCalled = true;
        }
    };

    // Call the hook directly to verify behavior
    $importer->beforeSave(new \App\Models\Contact, ['name' => 'Test']);

    expect($importer->hookCalled)->toBeTrue();
});
```

## Fake Import Manager

`Tapix::fake()` provides a lightweight way to assert that imports were triggered without actually processing rows. This is useful for testing controllers, Livewire components, or any code that triggers an import.

### Setting Up

Call `Tapix::fake()` in your test setup. Always call `Tapix::clearFake()` in `afterEach` to reset state between tests:

```php
use Tapix\Core\Tapix;

beforeEach(function () {
    Tapix::fake();
});

afterEach(function () {
    Tapix::clearFake();
});
```

### Asserting Imports

```php
use App\Importers\ContactImporter;
use App\Importers\ProductImporter;
use Tapix\Core\Tapix;

it('triggers a contact import', function () {
    // ... trigger import logic ...

    Tapix::assertImported(ContactImporter::class);
    Tapix::assertNotImported(ProductImporter::class);
});

it('imports the expected number of times', function () {
    // ... trigger two imports ...

    Tapix::assertImportedCount(2);
});
```

### Available Assertions

| Method | Description |
|--------|-------------|
| `Tapix::assertImported(ImporterClass::class)` | Assert the importer was used at least once. |
| `Tapix::assertNotImported(ImporterClass::class)` | Assert the importer was never used. |
| `Tapix::assertImportedCount(2)` | Assert the total number of imports triggered. |

## Import Test Results

`ImportTestResult` provides fluent assertions for verifying row-level outcomes after running an import in tests:

```php
use Tapix\Core\Testing\ImportTestResult;

$result = new ImportTestResult(
    created: 5,
    updated: 2,
    skipped: 1,
    failed: 0,
);

$result
    ->assertCreated(5)
    ->assertUpdated(2)
    ->assertFailed(0)
    ->assertNoFailures();
```

### Available Assertions

| Method | Description |
|--------|-------------|
| `->assertCreated(int $expected)` | Assert the number of created rows. |
| `->assertUpdated(int $expected)` | Assert the number of updated rows. |
| `->assertFailed(int $expected)` | Assert the number of failed rows. |
| `->assertNoFailures()` | Assert zero failures and no failed row data. |

All assertions are fluent and return `$this`, so they can be chained.
