Tapix uses three Eloquent models to track the state of an import throughout its lifecycle. Table names are configured via `table_names` in `config/tapix.php` (defaults: `tapix_imports`, `tapix_failed_import_rows`).

## Import

`Tapix\Core\Models\Import`

The primary model representing an import session. Tracks the overall status, column mappings, and row counters.

### Properties

| Column | Type | Description |
|---|---|---|
| `id` | ULID | Primary key |
| `tenant_id` | nullable | Tenant scoping identifier |
| `user_id` | string | The user who initiated the import |
| `importer` | string | Fully qualified class name of the importer |
| `file_name` | string | Original uploaded file name |
| `status` | ImportStatus | Current import status |
| `headers` | JSON array | Detected CSV column headers |
| `column_mappings` | JSON array | User-defined column-to-field mappings |
| `total_rows` | integer | Total number of rows in the CSV |
| `created_rows` | integer | Rows that created new records |
| `updated_rows` | integer | Rows that updated existing records |
| `skipped_rows` | integer | Rows that were skipped |
| `failed_rows` | integer | Rows that failed during processing |
| `completed_at` | timestamp | When the import finished |
| `file_path` | nullable string | Path to the uploaded file in storage |
| `results` | JSON | Import results summary |

The `status` column uses the `ImportStatus` enum with the following values: `Uploading`, `Mapping`, `Reviewing`, `Previewing`, `Importing`, `Completed`, `Failed`.

### Relationships

- `user()` -- `BelongsTo` the user who initiated the import.
- `failedRows()` -- `HasMany` `FailedImportRow` records.

### Key Methods

- `columnMappings(): Collection<ColumnData>` -- Returns the column mappings as a hydrated collection of `ColumnData` objects.
- `getImporter(): Importer` -- Instantiates and returns the importer class.
- `transitionToImporting(): bool` -- Performs an atomic status transition to `Importing` using a cache lock to prevent race conditions.
- `storagePath(): string` -- Returns the filesystem path for this import's store directory.
- `setColumnMappings(array $mappings): void` -- Persists column mappings to the model.
- `getColumnMapping(string $source): ?ColumnData` -- Returns the mapping for a specific source column.
- `updateColumnMapping(string $source, array $data): void` -- Updates a single column mapping.

### Scopes

- `completed()` -- Filters imports with status `Completed`.
- `failed()` -- Filters imports with status `Failed`.

## ImportRow

`Tapix\Core\Store\ImportRow`

Represents a single row from the imported file. Unlike `Import` and `FailedImportRow`, this model does **not** live in your main database. Each import gets its own per-import SQLite file, managed by `ImportStore`.

::note
Import rows are stored in per-import SQLite databases at the path configured by `store.path` (default: `storage/app/tapix/{importId}/data.sqlite`). This architecture keeps row-level operations fast without impacting your main database.
::

### Accessing Rows

Use `ImportStore` to query rows for a specific import:

```php
use Tapix\Core\Store\ImportStore;

$store = ImportStore::load($import->id);
$rows = $store->query()->withErrors('email')->get();
$createCount = $store->query()->toCreate()->count();
```

### Properties

| Column | Type | Description |
|---|---|---|
| `row_number` | integer | **Primary key.** Position in the original file. |
| `raw_data` | JSON (Collection) | Original row data from the CSV |
| `validation` | JSON (Collection) | Validation errors per column |
| `corrections` | JSON (Collection) | User corrections per column |
| `skipped` | JSON (Collection) | Columns marked as skipped |
| `match_action` | RowMatchAction | Whether to create, update, or skip |
| `matched_id` | nullable | ID of the existing record (for updates) |
| `relationships` | JSON (array) | Resolved relationship matches |
| `processed` | boolean | Whether the row has been processed |

### Key Methods

- `getFinalValue(string $column)` -- Returns the corrected value if one exists, the raw value otherwise, or `null` if the column is skipped or has errors.
- `getFinalData(): array` -- Returns merged raw data with corrections applied.
- `hasErrors(): bool` -- Whether the row has any validation errors.
- `hasCorrections(): bool` -- Whether the row has any user corrections.
- `hasValidationError(string $column): bool` -- Whether a specific column has a validation error.
- `isValueSkipped(string $column): bool` -- Whether a specific column value was skipped.
- `isCreate()`, `isUpdate()`, `isSkip()` -- Check the row's match action.

### Scopes

- `withErrors($column)` -- Rows with validation errors on a specific column.
- `withCorrections($column)` -- Rows with user corrections on a specific column.
- `withSkipped($column)` -- Rows with skipped values on a specific column.
- `valid()` -- Rows without validation errors.
- `toCreate()` -- Rows with `Create` match action.
- `toUpdate()` -- Rows with `Update` match action.
- `toSkip()` -- Rows with `Skip` match action.
- `uniqueValuesFor($column)` -- Distinct values for a given column with counts.
- `searchValue($column, $search)` -- Search rows by column value.
- `forFilter(ReviewFilter, $column)` -- Apply a review filter for a specific column.

## ImportStore

`Tapix\Core\Store\ImportStore`

Manages the per-import SQLite database. This is the entry point for all row-level operations.

### Key Methods

| Method | Description |
|--------|-------------|
| `ImportStore::create(string $importId): self` | Create a new store for an import. |
| `ImportStore::load(string $importId): ?self` | Load an existing store. Returns `null` if not found. |
| `$store->query(): Builder` | Returns an Eloquent builder for `ImportRow` scoped to this store. |
| `$store->destroy(): void` | Delete the store and its SQLite file. |
| `$store->uploadToRemote(): void` | Upload the SQLite file to the configured remote disk. |
| `$store->cleanupRemote(): void` | Remove the SQLite file from the remote disk. |
| `$store->isRemote(): bool` | Whether a remote copy exists. |

## FailedImportRow

`Tapix\Core\Models\FailedImportRow`

Records rows that failed during the execution phase. Uses Laravel's `MassPrunable` trait for automatic cleanup.

### Properties

| Column | Type | Description |
|---|---|---|
| `id` | ULID | Primary key |
| `import_id` | string | Foreign key to the parent import |
| `data` | JSON | The row data that failed processing |
| `validation_error` | string | The error message |

### Relationships

- `import()` -- `BelongsTo` the parent `Import` record.

The `MassPrunable` trait defines a prunable query that removes rows older than the `cleanup_after_days` config value. Run `php artisan model:prune` to clean them up.

## Database Tables

Two tables are created by Tapix migrations in your main database:

1. `tapix_imports` -- Main import records.
2. `tapix_failed_import_rows` -- Rows that failed during execution.

Import row data is stored in per-import SQLite files, not in the main database. See the [ImportRow](#importrow) section above.
