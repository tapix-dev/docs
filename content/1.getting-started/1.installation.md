Tapix is split into two packages: `tapix/core` (the import engine) and `tapix/livewire` (the wizard UI + Filament plugin). Install only what you need.

## Requirements

### Core (engine)

- PHP 8.4 or higher
- Laravel 12 or higher
- A supported database: MySQL, PostgreSQL, or SQLite

### UI (if using the Livewire wizard)

- Livewire 4 or higher
- Tailwind CSS 4 or higher

### Optional: Filament panel integration

- Filament 5 or higher

::note
**Using Filament?** The first-party Filament v5 plugin lives in `tapix/livewire`. See the [Filament integration](/integration/filament) page for setup details after completing the base installation below.
::

## Installation

::steps

### Add the Tapix repository

Register the private Tapix Composer repository in your project:

```bash [Terminal]
composer config repositories.tapix composer https://tapix.dev/repo
```

When prompted, enter the credentials you received after purchasing your license.

### Install the package

Choose the package that fits your use case:

#### Engine only (headless/programmatic)

For projects using Inertia, custom UIs, API endpoints, or CLI-driven imports:

```bash [Terminal]
composer require tapix/core
```

#### With Livewire wizard

For projects wanting the 4-step import wizard UI. This automatically installs `tapix/core`:

```bash [Terminal]
composer require tapix/livewire
```

### Publish the configuration

Publish the Tapix configuration file:

```bash [Terminal]
php artisan vendor:publish --tag=tapix-config
```

This creates `config/tapix.php`. See the [Configuration](/getting-started/configuration) page for a full reference of available options.

### Run migrations

Tapix requires database tables to track imports and failed rows:

```bash [Terminal]
php artisan migrate
```

### Include Tapix styles

If you installed `tapix/livewire`, add the Tapix view path as a source in your main CSS file so Tailwind v4 includes the required utility classes:

```css [resources/css/app.css]
@source "../../vendor/tapix/livewire/resources/views";
```

::warning
Without this `@source` directive, the import wizard will render without proper styling.
::

::note
This step is only needed when using the Livewire wizard (`tapix/livewire`). The engine-only package (`tapix/core`) has no views.
::

### Create your first importer

Generate an importer class using the Artisan command:

```bash [Terminal]
php artisan make:tapix-importer ContactImporter
```

This creates a new importer at `app/Importers/ContactImporter.php`. Open the file and define your column mappings, validation rules, and processing logic. See the [Defining Importers](/importers/defining-importers) section for detailed guidance.

::

## Using the Wizard

The Livewire wizard requires `tapix/livewire`. After installation, you have two options for rendering it:

### Livewire Component (any Laravel app)

Embed the wizard in any Blade view using the Livewire component:

```blade
<livewire:tapix.wizard
    :importer-class="App\Importers\ContactImporter::class"
    :return-url="route('contacts.index')"
/>
```

See the [Livewire Component](/integration/livewire) page for all available props and configuration.

### Filament Plugin (Filament admin panels)

If your application uses Filament v5, register the `TapixPlugin` in your panel provider to get automatic page registration, navigation, and tenant support:

```php [app/Providers/Filament/AdminPanelProvider.php]
use Tapix\Livewire\Filament\TapixPlugin;

public function panel(Panel $panel): Panel
{
    return $panel
        ->plugins([
            TapixPlugin::make()
                ->discoverImporters(),
        ]);
}
```

See the [Filament integration](/integration/filament) page for full configuration options.

## Programmatic Usage

If you installed `tapix/core` without the wizard, use the `ImportManager` to run imports programmatically from artisan commands, API endpoints, or scheduled tasks:

```php
use Tapix\Core\Tapix;
use App\Importers\ContactImporter;

$manager = Tapix::import(ContactImporter::class);
$import = $manager->createFromFile('/path/to/contacts.csv');
$mappings = $manager->autoMap($import);
$batchIds = $manager->validate($import);
$executionBatchId = $manager->execute($import);
```

See the [Programmatic API](/integration/programmatic) page for the full reference and examples.

## Verification

After completing the installation steps, verify everything is working:

1. Navigate to the page where you embedded the Livewire wizard component (or your Filament admin panel if using the plugin).
2. Confirm the import wizard loads and displays the upload step.
3. Try uploading a small CSV file to verify file parsing works.

If the wizard does not appear, verify that:

- You have at least one importer class in your configured importer directory.
- You have run `php artisan migrate` to create the required tables.
- Your Tailwind build includes the Tapix content path.
- If using Filament: `TapixPlugin` is registered and panel cache is cleared (`php artisan filament:clear-cached-components`).
